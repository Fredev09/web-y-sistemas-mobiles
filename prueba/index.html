Mi primera pagina web

Introduccion

HTML (HyperText Markup Language) es el lenguaje base para crear y estructurar páginas web. Su función principal es organizar el contenido (títulos, párrafos, secciones y otros elementos) para que el navegador lo interprete y lo muestre de forma ordenada. A lo largo de los años, HTML ha ido cambiando para adaptarse a nuevas necesidades: mejores diseños, contenido multimedia, accesibilidad y compatibilidad con dispositivos modernos.

evolucion de html

Los primeros formatos de HTML surgieron a inicios de los años 90, cuando la web era principalmente texto y enlaces. En esas versiones iniciales, el objetivo era compartir información de manera simple, usando estructuras básicas como títulos y párrafos.

Con el crecimiento de Internet, la web comenzó a necesitar más que texto: listas, tablas, formularios y formas más organizadas de presentar información. En esa etapa, HTML fue ampliándose para permitir páginas más completas e interactivas, aunque muchas veces se mezclaba estructura con “estilos” dentro del mismo documento.

Con el tiempo, se volvió necesario separar responsabilidades: HTML se enfoca en estructura, CSS en diseño y JavaScript en comportamiento. Esto ayudó a que el desarrollo web fuera más ordenado, mantenible y adaptable.

Finalmente, con HTML5 se dio un salto importante: se incluyeron etiquetas más semánticas (como secciones y artículos), soporte nativo para audio y video, mejoras para formularios, y una base más sólida para aplicaciones web modernas. Actualmente, HTML sigue evolucionando para mejorar la accesibilidad, la compatibilidad entre navegadores y la experiencia en móviles.

Conclusion

Aunque HTML mantiene su propósito central de estructurar contenido, su evolución ha respondido a cómo cambió la web: de páginas simples a plataformas ricas en multimedia, accesibles y pensadas para múltiples dispositivos. Gracias a esto, hoy es posible crear sitios y aplicaciones más completas sin perder el orden y la claridad en la organización del contenido.

Reflexion

1. ¿Qué limitaciones notaste al trabajar solo con títulos y párrafos, y cómo crees que esas limitaciones impulsaron la evolución de HTML?

Las limitaciones son principalmente hacer algo atractivo a los usuarios, ya que por ejemplo no se pueden agregar estilos ni tampoco poner imagenes ni videos, por lo cual la pagina va a quedar como si fuera un documento de texto simple. Estas limitaciones impulsaron la evolución de HTML porque la web empezó a necesitar más organización, más interacción y más tipos de contenido. Por eso se añadieron nuevas etiquetas y características para cubrir necesidades como registrar datos, estructurar mejor la información, incluir multimedia y facilitar la creación de sitios modernos.

2. ¿Por qué crees que HTML ha evolucionado tanto si su objetivo principal sigue siendo estructurar contenido?

Yo creo que html ha evolucionado dado que actualmente existe una gran cantidad de dispositivos, ya no solo computadores, por lo cual el contenido se debe adaptar a los demas dispositivos y se utilizan mas contenido como imagenes y videos se hacen acciones, como por ejemplo comprar en una pagina, entonces ha evolucionado para facilitar y mejorar la estructuracion del contenido de forma mas organizada
